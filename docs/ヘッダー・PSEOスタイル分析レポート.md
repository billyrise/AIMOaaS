### 1.3 ヘッダー課題の整理

- **PSEO のみ「サイトの一部」であることが視覚・操作で伝わりにくい**  
  他ページは fixed ヘッダーで常にロゴ・ナビ・CTA が表示されるのに対し、PSEO はパンくず1行のみで、トップや他セクションへの導線が弱い。
- **リソース記事型はロゴがテキストのみ**で、メインLP・AIMO Standard・監査法人とアイコン有無が揃っていない。
- **ナビ項目のばらつき**  
  メインLPには「リソース」「実務ガイド」があるが、AIMO Standard 型・監査法人型・リソース記事型には「実務ガイド」（/ja/resources/pseo/）がなく、PSEO 一覧・記事への発見性が低い。
- **言語切替**  
  PSEO は日本語のみのため言語切替は不要だが、他言語サイトから ja の PSEO に来た場合の戻り先がヘッダーにない。

## 第2部：PSEO ページのスタイルが他とかけ離れている課題

### 2.1 比較の観点

他ページ（メインLP・AIMO Standard・監査法人・リソース記事）と PSEO を、**レイアウト・コンポーネント・視覚**で比較する。

---

### 2.2 レイアウト・ページ構造の差

| 観点 | 他ページ（メインLP / aimo-standard / 監査法人 / リソース記事） | PSEO ページ |
|------|--------------------------------------------------------------|-------------|
| **ヘッダー** | fixed のグローバルナビ（ロゴ・ナビ・CTA・言語） | パンくず1行のみ（static） |
| **メインの上余白** | `pt-24` または `pt-20`（fixed ヘッダー分） | `py-8` のみ（ヘッダー高さなし） |
| **ラッパー幅** | `container mx-auto px-6` ＋ `max-w-3xl` など | `max-w-4xl mx-auto px-4 py-8`（1本の div で全体） |
| **メイン内のパンくず** | 多くのリソースで「ホーム / リソース / 当該ページ」を main 内に表示 | ヘッダー兼パンくずで「AIMOaaS™ / 監査・証跡（実務）」のみ |
| **フッター** | ダーク（slate-900/950）、ロゴ・ナビ・著作権など | ライト、1行（ホーム \| AIMO Standard） |

→ **PSEO だけ「ヘッダーが固定でない」「メイン上余白が小さい」「フッターがライトでミニマル」** という構造になっており、同一サイト内の「記事」としての一体感が弱い。

---

### 2.3 コンポーネント・UI の差

| 観点 | 他ページ | PSEO ページ |
|------|----------|-------------|
| **body 背景** | `bg-slate-50 text-slate-800`（他と共通のライト） | `bg-slate-50 text-slate-900`（やや濃いテキスト） |
| **フォント** | Noto Sans JP（300,400,500,700,900 などウェイト多め） | Noto Sans JP（400,500,700 のみ） |
| **追加 CSS** | hero-bg, grid-bg, glass-card, モバイルメニュー等 | `.pseo-prose` のみ（表・見出し・行間） |
| **CTA** | ナビ横にボタン、セクション内に CTA ブロック | 本文中の CTA ブロックのみ（ヘッダーに CTA なし） |
| **アイコン** | Lucide（shield-check 等）使用 | アイコンなし |
| **言語・hreflang** | 多言語ページでは alternate あり | 日本語のみ、alternate なし（想定どおり） |

→ **PSEO は「記事テンプレート」用の最小スタイルに特化しており、サイト共通のナビ・ブランド・CTA がヘッダーにない。**

---

### 2.4 視覚・トーンの差

| 観点 | 他ページ | PSEO ページ |
|------|----------|-------------|
| **トップの印象** | ヒーロー／タイトル帯／ナビで「サービスサイト」と分かる | タイトル＋TL;DR＋CTA で「記事・ガイド」寄り |
| **セクション構成** | ヒーロー、特徴、プラン、FAQ などブロックが豊富 | TL;DR → 本文 → mid-cta → trust → FAQ → bottom-cta → 次に読む → フッター |
| **フッターの色** | ダーク（slate-900/950）でサイト全体のトーンと統一 | ライト＋細いボーダーのみで「別サイトの記事」のように見える |

→ **ユーザーが「同じ AIMOaaS サイト内を移動している」と感じにくく、特にトップやリソース一覧から PSEO 記事に来た場合に違和感が出やすい。**

---

### 2.5 PSEO スタイルがかけ離れている主因

1. **テンプレートの目的の違い**  
   PSEO は「SEO・コンテンツ量・構造化データ」を優先した**記事専用テンプレート**（`templates/pseo/page.html`）で生成されている。他ページは手書きまたは別テンプレートで、**マーケ・ブランド・ナビ**を前提にしている。
2. **ヘッダー・フッターの未共通化**  
   共通のヘッダー・フッター partial がなく、PSEO テンプレートには「パンくず＋ミニマルフッター」だけが入っている。
3. **デザイン方針の未統一**  
   サイト全体で「fixed ヘッダー＋ダークフッター」を標準とするかどうかが明文化されておらず、PSEO だけ別ラインで作られた結果、スタイル差が固定化している。

---

### 2.6 課題まとめ（PSEO スタイル）

| 課題 | 内容 |
|------|------|
| **ナビ・ブランドの欠如** | PSEO では fixed ヘッダーがなく、ロゴ・グローバルナビ・お問い合わせが常時見えない。 |
| **フッターの不一致** | 他はダークでロゴ・ナビ・著作権があるのに対し、PSEO はライトで1行のみ（フッター分析レポートの「5・6」に該当）。 |
| **レイアウト差** | 上余白・ラッパー・パンくずの有無が他と異なり、「同じサイト」感が弱い。 |
| **導線の弱さ** | 実務ガイド一覧・トップ・お問い合わせへの露出が他ページより少ない。 |
| **SEO・UX のトレードオフ** | コンテンツ量と読みやすさを優先した結果、サイト内回遊・信頼シグナル（共通UI）が弱くなっている。 |

---

## 第3部：推奨アクション（ヘッダー・PSEO スタイル）

### 3.1 ヘッダー

- **共通ヘッダー partial の作成**  
  メインLP型をベースに、言語・現在地・CTA 文言だけを変数化した「共通ヘッダー」を用意する。
- **全セクションでの採用**  
  メインLP・AIMO Standard・監査法人・リソース記事・**PSEO（一覧＋個別記事）** のすべてで、この共通ヘッダー（fixed）を表示する。
- **PSEO での必須項目**  
  少なくとも「ロゴ（ホームへのリンク）」「実務ガイド一覧（/ja/resources/pseo/）」「AIMO Standard」「お問い合わせ」をヘッダーに含め、実務ガイド記事であることが分かるようにする。

### 3.2 PSEO スタイル

- **ヘッダー・フッターを他ページと共通化する**  
  上記の共通ヘッダー＋フッター統一レポートで推奨した共通フッターを PSEO テンプレートにも組み込む。これにより「固定ヘッダー＋ダークフッター」が PSEO にも適用され、スタイルの差が縮小する。
- **body・メインの余白を他に合わせる**  
  共通ヘッダーを入れたうえで、`main` に `pt-24`（または他ページと同一の pt）を入れ、ラッパー幅（例: max-w-3xl / max-w-4xl）は他リソース記事と揃えるか、意図的に広くするかを方針化する。
- **フォント・カラー**  
  body の `text-slate-800` / `text-slate-900` は他と統一し、追加 CSS は `.pseo-prose` に限定したまま、ヘッダー・フッターは共通スタイルに任せる。

上記により、**ヘッダー・フッター・レイアウトの一貫性**が取りやすくなり、PSEO ページの「他とかけ離れている」という課題が解消される方向になる。
