## 3. SEO の観点で「同一表示」がベストな理由

### 推奨: **サイト全体でフッターを共通化し、同一（または言語・セクションに応じた最小バリエーション）にする**

#### 理由

1. **サイト構造・内部リンクの一貫性**
   - クローラーはフッターのリンクでサイト構造を把握する。ページごとにフッターが違うと、「どのページが重要か」「トップ・カテゴリ・実務ガイドの関係」が伝わりにくい。
   - 主要なナビ（ホーム・AIMO Standard・実務ガイド・お問い合わせ・運営会社）を**全ページで同じ方針**で出せば、重要ページへのリンク重み（内部リンク）が安定し、インデックス・ランキングの安定につながる。

2. **ユーザー体験と直帰率**
   - どのページからでも「ホームに戻る」「実務ガイド一覧」「お問い合わせ」に同じ操作で到達できると、直帰率が下がり、セッションあたりの閲覧ページ数が増えやすい。これはエンゲージメント指標として検索品質にも間接的に影響する。

3. **重複コンテンツ・テンプレートの一貫性**
   - フッターがバラバラだと、将来的に「フッターだけ差し替えた類似ページ」とみなされるリスクは小さいが、「サイトの一貫性」の観点では弱い。共通フッターにすると、テンプレート・コンポーネントが一元化され、メンテナンスと一貫性の両立がしやすい。

4. **PSEO 記事の扱い**
   - 現在、PSEO 記事（約100ページ）は**フッターが最小**（ホーム｜AIMO Standard のみ、ライト背景）。SEO的には「トップや実務ガイド一覧へのリンクはある」が、「お問い合わせ」「運営会社」「リソース一覧」などへの導線が欠けている。
   - 他セクションと**同じナビ＋著作権・運営会社**を出せば、実務ガイド群がサイトの一部であることが明確になり、サイト全体のテーマ性（AIガバナンス・監査・証跡）の伝達が強くなる。

5. **クロール効率**
   - フッターを共通化すると、重要なURL（トップ、実務ガイド、お問い合わせなど）が全ページから張られる。クローラーがどの入口で来ても、主要ページへ効率的にたどりつける。

## 4. 具体的な推奨

- **共通フッターの内容（推奨最小セット）**
  - ロゴ（ホームへのリンク）
  - ホーム / AIMO Standard / 実務ガイド（`/ja/resources/pseo/` 等）/ お問い合わせ（`/#contact` 等）/ 運営会社（riseby.net）
  - 著作権および「AIMOaaS™ は ○○ が提供」の一文（言語別で1パターンずつ）

- **デザイン**
  - 全ページ**同一のスタイル**（例: メインLP型のダーク `bg-slate-950`）に揃えると、見た目とSEOの両方で一貫性が保てる。PSEO のみライト背景にしたい場合は、**リンクと文言だけ共通**にし、クラスで見た目だけ調整する方法もある。

- **実装**
  - 静的HTMLのため、**共通フッターを partial やスニペットとして切り出し**、ビルド時やスクリプトで各ページに差し込む形にすると、今後の変更が一箇所で済む。
