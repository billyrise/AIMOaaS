# AIMOaaS サイト デザインガイドライン

本ドキュメントは、トップページおよび主要ページ（メインLP・AIMO Standard・監査法人・リソース記事）の現在のデザインを**正（canonical）**として、サイト全体で守るデザインのルールを定義する。  
`/resources/pseo/`（PSEO）配下のページは、このガイドラインに従って他ページと統一すること。

**関連ドキュメント**
- [ヘッダー・PSEOスタイル分析レポート](./ヘッダー・PSEOスタイル分析レポート.md)
- [フッター統一レポート](./フッター統一レポート.md)
- [ヘッダー・フッター統一修正計画](./ヘッダー・フッター統一修正計画.md)

## 1. デザインの原則

- **一貫性**: どのページからでも「同じ AIMOaaS サイト内にいる」と認識できること。
- **アクセシビリティ**: コントラスト・フォーカス・セマンティクスを守ること。
- **ブランド**: ロゴ・CTA・ナビが常時利用可能であり、信頼と導線を損なわないこと。

## 2. カラーパレット

### 2.1 基本（Tailwind クラスで指定）

| 用途 | クラス | 補足 |
|------|--------|------|
| **body 背景** | `bg-slate-50` | 全ページ共通。薄いグレー。 |
| **本文テキスト** | `text-slate-800` | 本文・リード文。PSEO では `text-slate-900` が使われている場合は `text-slate-800` に統一推奨。 |
| **見出し・強調** | `text-slate-900` | H1・H2・重要なラベル。 |
| **補助テキスト** | `text-slate-600` / `text-slate-500` | キャプション・メタ・パンくずの現在地以外。 |
| **ボーダー** | `border-slate-200` | ヘッダー下・カード・区切り線。 |

| **フッター境界** | `border-t border-slate-900` | フッター上線。 |
| **フッター文言** | `text-slate-400` / `text-slate-500` | リンク・説明・著作権。 |

### 2.2 ブランド・アクション色（Indigo）

| 用途 | クラス | 補足 |
|------|--------|------|
| **ロゴ「aaS」部分・リンクホバー・アクセント** | `text-indigo-600` | ナビ hover、パンくずリンク。 |
| **プライマリ CTA 背景** | `bg-indigo-600` | ボタン・ナビ CTA。 |

### 2.3 その他（状態・装飾）

- **成功・チェック**: `text-green-400` / `bg-green-100` 等（ヒーローのチェックマーク等）。
- **警告・エラー**: `text-red-500` / `bg-red-100` 等。
- **グラデーション（ヒーロー）**: `from-indigo-400 to-pink-400`（テキスト）、`from-indigo-500` / `to-pink-600`（背景・装飾）。

## 4. レイアウト・余白・幅

### 4.1 共通ルール

- **ヘッダー高さ**: `h-16`（64px）。固定ヘッダー使用時はメインの上パディングでオフセットする。
- **メイン上余白（fixed ヘッダーあり）**: `pt-24`（96px）。全ページ統一（ランディング・リソース・PSEO とも）。
- **コンテナ**: `container mx-auto px-4 md:px-6`（ヘッダー・フッター）、`container mx-auto px-6`（メイン内）。
- **コンテンツ幅（記事・リソース）**: `max-w-3xl` または `max-w-4xl`。リソース記事は `max-w-3xl`、PSEO は `max-w-4xl` を使用可。サイト全体でどちらを標準とするか方針を決め、PSEO も揃える。

### 4.2 ブレークポイント（Tailwind 既定）

- `sm`: 640px
- `md`: 768px
- `lg`: 1024px
- `xl`: 1280px

**ヘッダー・ナビの表示切替**
- **メインLP・静的ページ**: PC でナビを表示する breakpoint は `xl`（1280px）。モバイルメニューは `xl:hidden`。
- **PSEO**: 現状 `md` でナビ表示のため、他と揃える場合は `xl` に統一する。

## 5. ヘッダー（共通仕様）

全ページ（トップ・AIMO Standard・監査法人・リソース・**PSEO 一覧・PSEO 個別記事**）で、以下を満たす**固定ヘッダー**を使用する。

### 5.1 外観・構造

- **ラッパー**: `fixed w-full z-50 bg-white/90 backdrop-blur-md border-b border-slate-200`
- **内側**: `container mx-auto px-4 md:px-6 h-16 flex items-center justify-between gap-3`
- **ロゴ**: 必ず **アイコン + テキスト**。`<i data-lucide="shield-check" class="text-indigo-600 w-8 h-8"></i>` + `AIMO<span class="text-indigo-600">aaS</span>™`。リンクはホーム（言語別トップ）へ。

### 5.2 ナビ（PC）

- **表示 breakpoint**: `xl` で表示（`hidden xl:flex`）。PSEO では `md` のままの場合は、他と揃えるなら `xl` に変更。
- **スタイル**: `text-sm font-medium text-slate-600`、`hover:text-indigo-600 transition`。
- **含めるリンク（最低限）**: ホーム、AIMO Standard、実務ガイド（`/ja/resources/pseo/` 等）、お問い合わせ（または CTA）。多言語ページでは言語切替（ドロップダウンまたは 1 リンク）を含める。

### 5.3 CTA ボタン

- **スタイル**: `bg-indigo-600 hover:bg-indigo-700 text-white px-4 py-2.5 rounded-full text-sm font-bold transition shadow-lg hover:shadow-indigo-500/30`
- **表示**: `hidden sm:inline-flex` で PC・タブレットでは表示。PSEO でも「お問い合わせ」等の CTA を同じスタイルで表示する。

### 5.4 モバイルメニュー

- **トリガー**: `xl:hidden` でハンバーガーボタン表示。`aria-label`・`aria-controls="mobile-menu"`・`aria-expanded` を適切に指定。
- **パネル**: `#mobile-menu`、`fixed inset-x-0 top-16 bottom-0 z-40`、`border-t border-slate-200/80`、背景は `landing.css` に合わせて `#ffffff` またはグラデーション。
- **中身**: ナビリンク（`mobile-nav-link` 相当）、CTA（`mobile-cta` 相当）、必要なら言語切替。
- **スタイル統一**: メインLPで使っている `landing.css` の `#mobile-menu`・`.mobile-menu-inner`・`.mobile-nav-link`・`.mobile-cta`・`.mobile-lang-label`・`.mobile-lang-grid` を標準とし、PSEO のモバイルメニューも同じクラス・見た目に揃える（背景・パディング・CTA のグラデーション・言語グリッドの有無はセクションに応じて調整可）。

## 6. フッター（共通仕様）

全ページで**同一のダークフッター**を使用する。

### 6.1 外観・構造

- **ラッパー**: `bg-slate-950 text-slate-400 py-12 border-t border-slate-900`
- **内側**: `container mx-auto px-6`、`flex flex-col md:flex-row justify-between items-center gap-6`

### 6.2 内容（最低限）

- **ロゴ**: ホームへのリンク。`<i data-lucide="shield" class="w-6 h-6 text-slate-600"></i>` + `AIMOaaS™`。`hover:text-white transition`。
- **ナビ**: ホーム、AIMO Standard、実務ガイド（PSEO 一覧）、お問い合わせ、運営会社（riseby.net）。`flex flex-wrap gap-6 text-sm`、`hover:text-white transition`。
- **著作権・提供者**: `text-xs text-slate-500`。AIMO 説明文 + 「AIMOaaS™ は ○○ が提供」+ © 年 AIMOaaS™。

PSEO のみライトフッターや 1 行リンクにしない。必ず上記ダークフッターと同一とする。

## 8. 本文・プロース（リソース・PSEO）

### 8.1 リソース記事（ai-governance-guide 等）

- **ラッパー**: `pt-24 pb-16` の `<main>`、その中で `container mx-auto px-6 max-w-3xl`。
- **パンくず**: `text-sm text-slate-500 mb-8`。`/` 区切り、リンクは `hover:text-indigo-600`。
- **本文**: `prose prose-slate max-w-none`。H2 は `text-xl font-bold text-slate-900 mt-8 mb-3` 等で明示しても可。
- **関連リンク**: `mt-12 pt-8 border-t border-slate-200`、`text-sm text-slate-500` + リンク一覧。

### 8.2 PSEO 記事

- **ラッパー**: 固定ヘッダー使用時は `pt-24 pb-8` のメイン、`max-w-4xl mx-auto px-4` でよいが、`pt-24` は必須。
- **パンくず**: メイン内に `mb-6 text-sm text-slate-500`。AIMOaaS™ / 監査・証跡（実務） / 当該記事。
- **TL;DR**: `.tldr-box` などで `p-5 bg-white rounded-lg border border-slate-200 shadow-sm`。本文は `.pseo-prose`（`main.css` の .pseo-prose で表・見出し・行間を定義）。
- **CTA ブロック**: `.pseo-cta`。プライマリボタン色・形状は上記に合わせる。中程・下部の CTA は `text-center w-full` でボタンと補足文を中央揃えにする。
- **中程以降のセクション幅**: mid-cta・役割分担の整理（trust-block）・よくある質問・bottom-cta・次に読むは、いずれも `w-full` を付け、上部本文と同じ `max-w-4xl` エリア内に表示する（テンプレート `templates/pseo/page.html` およびパーシャルで統一）。
- **内部リンクブロック**: 「次に読む」など。H2 + リンクリスト。スタイルは他セクションとトーンを合わせる。セクションに `w-full` を付ける。

### 8.3 PSEO 本文の表（視認性）

監査・法人向けの読みやすさのため、`.pseo-prose` 内の表は以下を満たす（`src/input.css` で定義）。

- **枠線**: セル・ヘッダーに `1px solid #cbd5e1`（slate-300）。薄い色は使わない。
- **ヘッダー**: 背景 `#f1f5f9`、`font-weight: 600`、下線 `2px solid #94a3b8` で列の区切りを明確に。
- **ゼブラストライプ**: `tbody tr:nth-child(even)` に `#f8fafc` を付け、行の追いやすさを確保。
- **ホバー**: `tbody tr:hover` で `#f1f5f9` を指定。
- **ラッパー**: `.table-wrap` に外枠・角丸・軽いシャドウを付け、表のブロックを明確に。
- **余白**: セルは `padding: 0.75rem 1rem`、行間・フォントサイズで可読性を確保。

## 10. CSS・アセットの読み込み

### 10.1 必須

- **全ページ**: `/assets/css/main.css`（Tailwind ビルド + .pseo-prose 等）。
- **トップ・メインLP・AIMO Standard・監査法人など**: `/assets/css/landing.css`（hero-bg, grid-bg, モバイルメニュー用）。
- **リソース記事・PSEO**: ヘッダーでモバイルメニューのスタイルを `landing.css` と揃える場合は `landing.css` も読み込む。または、PSEO 用に同じ見た目を `main.css` に含めてもよい。

### 10.2 フォント

- 英語: `Inter`（Google Fonts）。
- 日本語: `Noto Sans JP`（Google Fonts）。
- `<head>` で `preconnect` + `link` + `body` の `font-family` を指定。

### 10.3 アイコン

- **Lucide**: `/assets/js/lucide.min.js` を body 末尾で読み込み、`lucide.createIcons()` を実行。
- ヘッダー・フッターのロゴは `data-lucide="shield-check"` / `data-lucide="shield"` を使用。PSEO でも同じアイコンを付ける。

- [ ] **ヘッダー**: fixed、`bg-white/90 backdrop-blur-md border-b border-slate-200`、`h-16`。
- [ ] **ロゴ**: shield-check アイコン + AIMOaaS™、ホームへのリンク。

- [ ] **モバイルメニュー**: `xl:hidden`、`top-16`、中身は `.mobile-menu-inner`・`.mobile-nav-link`・`.mobile-cta` 等で landing と同様の見た目。
- [ ] **メイン上余白**: `pt-24`。
- [ ] **body**: `bg-slate-50 text-slate-800`（`text-slate-900` のみの場合は `text-slate-800` に統一推奨）。

- [ ] **landing.css**: モバイルメニューを landing と同一にする場合は PSEO ページでも `landing.css` を読み込む。
- [ ] **Lucide**: ヘッダー・フッターでアイコンを表示するため、`lucide.min.js` と `createIcons()` を実行。

## 12. 禁止事項・注意

- **ヘッダーを固定しない**: 全ページで fixed ヘッダーを使う。パンくずのみの「ヘッダー」にしない。
- **フッターをライトにする**: PSEO のみライト・1 行フッターにしない。必ずダーク共通フッター。
- **ロゴからアイコンを外す**: ヘッダー・フッターとも shield-check / shield を付ける。
- **ナビから「実務ガイド」を外す**: 日本語等では実務ガイド（PSEO 一覧）へのリンクを入れる。
- **CTA をヘッダーから外す**: お問い合わせ（または無料相談）は全ページでヘッダーに表示する。
- **body の text 色をバラバラにする**: `text-slate-800` を標準とし、PSEO の `text-slate-900` のみの場合は統一を検討する。

## 14. PSEO レイアウト一括適用（既存 HTML 向け）

新規生成以外で、既に存在する PSEO の静的 HTML をガイドラインに合わせて更新する場合は、以下を実行する。

```bash
cd scripts/pseo && npm run apply-design
```

- **固定ヘッダーあり**: ヘッダーを新 partial に差し替え、`landing.css` 追加、`body` を `text-slate-800`、Lucide を `/assets/js/lucide.min.js`（defer）に統一。
- **旧レイアウト（パンくずのみ）**: 共通固定ヘッダー・ダークフッターを注入、Tailwind CDN をやめて `main.css` + `landing.css` に変更、`pt-24`・パンくず・Lucide を上記と同様に適用。

実施後は「サイト軽量化・パフォーマンス最適化計画」の共通化チェックリスト（CSS 同一 URL・Lucide defer 等）も満たす。
